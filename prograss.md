# MineGuard App 整体设计方案

### 一、 App整体构造

App将采用主流的 **“底部标签栏 (Bottom Tab Bar)”** 导航模式，结合 **“导航栈 (Navigation Stack)”** 进行页面跳转。这种结构清晰，符合移动用户的使用习惯。

*   **核心导航 (5个主标签页)**:
    1.  **首页**: 数据统计的核心看板。
    2.  **报警**: 报警信息的列表与管理中心。
    3.  **预览**: 实时视频监控的入口。
    4.  **状态**: 整个系统设备健康度的监控中心。
    5.  **我的**: 用户个人信息、设置及其他功能的集合。
*   **全局功能**:
    *   **智能对话**: 以悬浮按钮 (Floating Action Button) 的形式常驻在主要页面，方便随时调用。
    *   **登录/权限**: 在App启动时进行权限验证，未登录则强制跳转至登录页。

### 二、 整体页面数量及功能

预计需要 **10个** 核心页面来承载 `readme.md` 中描述的所有功能。

1.  **登录页**
    *   **功能**: 提供用户账号密码输入、登录验证。

2.  **首页 (数据统计)**
    *   **功能**:
        *   顶部展示关键数据卡片：报警总数、已处理数、设备总数、在线设备数。
        *   下方集成多个图表组件，可通过页签或滚动切换查看：报警趋势、场景报警统计、设备报警统计、报警类型统计。
        *   提供全局时间筛选器（今日、近7天、近30天）。

3.  **报警列表页**
    *   **功能**:
        *   以信息卡片的形式展示报警列表，包含图片缩略图、类型、设备、时间、状态等。
        *   顶部提供筛选/搜索栏，支持按时间、设备、类型、场景、状态等维度进行过滤。
        *   支持下拉刷新和上拉加载更多。
        *   点击卡片跳转至报警详情页。

4.  **报警详情页**
    *   **功能**:
        *   展示高清报警图片或视频录像。
        *   提供“保存到相册”功能。
        *   下方提供表单，供用户填写处理意见并提交。

5.  **设备列表页 (预览)**
    *   **功能**:
        *   以可折叠的列表形式，按“区域”对设备进行分组。
        *   显示每个设备的名称和在线状态。
        *   提供“多画面预览”入口。
        *   点击单个设备跳转至“单路预览页”。

6.  **单路预览页**
    *   **功能**:
        *   全屏或大窗口实时播放指定设备的视频流。
        *   底部提供工具栏：截图、开始/停止录像、语音对讲。

7.  **多画面预览页**
    *   **功能**:
        *   提供2x2、3x3等网格布局选项。
        *   在网格中同时播放多个设备的实时视频流。
        *   支持点击单个小窗口放大或进入单路预览模式。

8.  **系统状态页**
    *   **功能**:
        *   采用层级列表，清晰展示“服务器 -> 边缘计算单元 -> 摄像机”的树状结构。
        *   用不同颜色的状态图标（如绿点表示在线，红点表示离线）标识每个单元的健康状况。

9.  **我的页面**
    *   **功能**:
        *   显示用户头像、姓名和角色（管理员、操作员等）。
        *   提供功能入口：通知设置、修改密码、关于App、退出登录等。

10. **智能对话页**
    *   **功能**:
        *   标准的聊天界面。
        *   底部为文本输入框和发送按钮。
        *   上方为对话记录区，展示用户提问和AI的回答。

### 三、 前端CSS设计风格建议

考虑到工业/安防场景的特殊性，建议采用 **“深色模式优先 (Dark Mode First)”** 的现代工业风设计。

*   **色彩体系**:
    *   **主背景色**: 深邃的炭黑或深蓝灰色 (`#121212`, `#1F2937`)，减少弱光环境下的视觉刺激。
    *   **主内容色**: 白色或亮灰色 (`#FFFFFF`, `#E5E7EB`)，保证高对比度，文字清晰易读。
    *   **主强调色**: 科技蓝 (`#3B82F6`)，用于按钮、图标、图表线条和当前选中的状态，体现科技感。
    *   **辅助色**:
        *   **成功/在线**: 绿色 (`#10B981`)。
        *   **警告/待处理**: 橙色 (`#F59E0B`)。
        *   **危险/离线/报警**: 红色 (`#EF4444`)。

*   **字体与图标**:
    *   **字体**: 使用系统默认的无衬线字体（如 HarmonyOS Sans, PingFang SC, Roboto），字号稍大，确保可读性。
    *   **图标**: 采用线性、简约风格的图标（Iconfont），图标尺寸要足够大，便于点击。

*   **布局与组件**:
    *   **卡片 (Card)**: 使用带有圆角和轻微描边的卡片来承载内容块，与深色背景形成区隔。
    *   **按钮 (Button)**: 实体按钮要有明确的点击反馈（如颜色加深），尺寸要大，符合触摸操作习惯。
    *   **间距**: 保持元素之间足够的间距，避免界面显得拥挤，让布局更有呼吸感。
    *   **响应式**: 关键操作按钮和信息应置于屏幕下半部分，方便单手操作。
